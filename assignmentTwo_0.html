Week One: Making a Website

I was initially trained on the shaper by Madison, first making a design in Adobe Illustrator, then filling it with the shaper-appropriate colors, going through the safety training, learning about the different shaper bits, taping down my board, following instructions to carve out a coaster, and then sanding down the coaster's rough edges at the end.

Our wonderful TA Tuna who should totally be paid a ton more helped me set up a file location using Command Prompt. We cloned the repository using the `git clone https://github.com/roman-hyacinths/roman-hyacinths.github.io.git` where the URL at the end is different for each repository. Cloning allowed me to create a linked copy of the repository, which then allowed me to start pushing and pulling code. Pushing is used after making changes in the local folder of the repository, as it updates the cloud repository. Pulling is similar to pushing, but instead of sending files from the local folder to cloud, it copies the files from the cloud to local. This allows users to undo mistakes or download code that others have updated.

I then set up an index.html file, wrote "hello world," and pushed the changes!

(I had initially entered a different name for the file — "julialee-papastavros.github.io" — but I then realized that since my username was roman-hyacinths, this was an incorrect URL. I had to go back and change my file and repository name to "roman-hyacinths.github.io", and that fixed the error.)

HTML, CSS, and JavaScript are scripting/programming languages used in part for building websites. HTML is mostly used for the page's structure, whereas CSS is used for styles. JavaScript is generally used for more complex features, and I didn't use it for my website. I already had some existing knowledge of HTML, CSS, and JavaScript, so I decided to go for a template instead of going through the headache of coding something from scratch. Tuna suggested I use his code, which I decided to do after judging his website design a tiny bit.
